  WHAT THIS FILE IS
  - A small set of reusable summary "families" (A, B, C…). Each family is one structural pattern —
    a fixed STEM (who you are + the proof point you anchor on) plus swappable CLOSERS (the
    role-specific "what I want to do now" line). You tailor a summary by picking a family and
    swapping its closer to the job's domain.
  - The point of families is reuse: a handful of proven shapes cover most roles, so generation
    starts from voice that already works instead of from scratch each time.
  - For the deeper corpus (every finalized summary you've actually submitted, with notes), see
    `05a-summary-library.md`. That file is learning ground-truth; THIS file is the quick menu.

  HOW INTAKE FILLS IT
  - Intake interviews you for 2–3 distinct summary STEMS that fit the role types you target
    (e.g. one consumer-product stem, one platform/B2B stem, one mission-led stem), plus a few
    CLOSERS per stem for the domains you apply into. It writes them into the family slots below
    and fills the Quick Selection Guide so the agent can map a role type → a family in one step.
  - {{PLACEHOLDERS}} and <!-- intake: ... --> comments mark exactly what to fill. Start with 2–3
    families; add more over time as `05a-summary-library.md` reveals new proven shapes.

  CRAFT RULES (keep these — they are reusable regardless of who you are)
  - ~3 sentences. First person. Use contractions. Read it aloud: it should sound like you talking,
    not a corporate bio.
  - Conversational and human, not stiff or jargon-heavy. Higher-signal and warmer than the rest of
    the resume.
  - End on a personal-pull closer — the "what I want to build / who I want to help now" line, tuned
    to the role's domain. This is the part you swap per job.
  - Match the approximate length of your proven examples; the summary block has limited, roughly
    fixed space on the page.
