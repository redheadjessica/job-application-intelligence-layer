## Purpose
This file is the structured evidence bank for {{CANDIDATE_NAME — the user's full name}}'s experience. It holds reusable experience themes, role-level bullet banks, story anchors, and proof-ready framing that can be pulled into tailored resumes, cover letters, interview prep, and application forms.

This is not a final resume. It is the raw material behind the resume.

## ⭐ Quantitative Proof Is Precious (hard rule)
Most candidates have **fewer** clean quantitative measures than they think, so every defensible number carries outsized value.
- **Never over-trim concrete proof.** Do not drop a metric-bearing or concrete proof point for "space." Swap something softer instead. This is a hard line.
- **Lean toward including ANY number the user is comfortable stating publicly** whenever it fits: team size, activation, engagement, retention, company size, user counts, growth rates, conversion, contract value, etc. When unsure whether a number helps, include it.
- **Canonize newly-approved numbers.** Any time a new concrete number is approved/used in a finalized resume, record it in the **Metrics Master List** (Section 4) and the relevant role's Proof/Metrics so it's reusable later. The number bank should only grow.
- Still factual: only numbers the user is comfortable stating publicly and can defend.

## Core Rules
- Keep entries factual and defensible.
- Separate broad claims from specific proof.
- Prefer modular bullets that can be adapted for different roles.
- Include context, action, and impact where possible.
- When details are uncertain, mark them for verification instead of overstating.
- Use Core Bullets as defaults.
- Use Supplemental / Variant Bullets only when they improve fit for the target role.
- Keep proof and metrics separate from narrative claims where possible.
- Treat metrics as reusable only when they are already supported by prior resume usage or clearly tied to known experience.
- If a number or claim is useful but not fully confirmed, mark it for verification before using it in a final resume.
- Prefer metrics that reinforce a specific bullet over standalone "impressive" numbers with weak context.
- Do not duplicate or stack too many metrics in one section unless they materially strengthen fit.

## ⭐ Title & Seniority Claims — State Precisely, Never Overclaim
When a role's wording touches a title, ordinal, or scope claim ("first ___", "head of ___", "led the ___ team", "founder"), state it **exactly as precisely as it is true** and no further.

## Bullet Selection Rules
- Start with Core Bullets.
- Swap in Supplemental / Variant Bullets only when they materially improve match with the target role.
- Avoid overloading any one role with too many bullets unless it is the anchor role for the application.
- Prefer recent, strategic, and role-matched evidence.

## Guardrails
- Do not inflate domain depth beyond what the role actually supports.
- Do not overuse early-career or founder titles in a way that distorts current seniority.
- Keep claims tied to shipped work, operating scope, and defensible outcomes.

## Proof and Metrics Usage
- Use metrics to support the strongest relevant claims, not to overload every bullet.
- Prefer metrics that match the target role's priorities (growth, monetization, scale, leadership, platform, etc.).
- When a metric is less certain or rarely used, keep it out of final resume output unless explicitly reviewed.
- If space is tight, prioritize the clearest and most credible proof over total quantity.
